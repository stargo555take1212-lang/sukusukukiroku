# Google Apps Script 連携セットアップ

このアプリのデータを Google スプレッドシートに保存し、夫婦二人の端末で共有するための手順です。

## 1. スプレッドシートを作る

1. Google スプレッドシートで新規シートを作成する（名前は何でもOK。例:「すくすくノート データ」）。
2. シートの中身は空のままでOK（アプリが自動でシートを作成します）。

## 2. Apps Script を貼り付ける

1. スプレッドシートのメニューから「拡張機能」→「Apps Script」を開く。
2. デフォルトで開かれる `コード.gs` の中身を全部削除し、このリポジトリの [Code.gs](Code.gs) の内容を全部貼り付ける。
3. 上部の保存アイコン（💾）でプロジェクトを保存する。

## 3. ウェブアプリとしてデプロイする

1. 右上の「デプロイ」→「新しいデプロイ」をクリック。
2. 種類の選択（歯車アイコン）で「ウェブアプリ」を選ぶ。
3. 以下の設定にする。
   - 説明: 任意（例: `sukusuku api v1`）
   - 次のユーザーとして実行: **自分**
   - アクセスできるユーザー: **全員**
4. 「デプロイ」をクリックし、Googleアカウントの権限確認が出たら許可する。
5. 発行された「ウェブアプリのURL」（`https://script.google.com/macros/s/.../exec` の形式）をコピーする。

## 4. アプリの設定画面にURLを登録する

1. すくすくノートを開き、下部ナビゲーションの「設定」を開く。
2. 「GAS ウェブアプリURL」欄に、3.で発行されたURLを貼り付けて「接続する」を押す。
3. 「接続に成功しました」と表示されればOK。
4. パートナーの端末でも同じ手順（設定画面にURLを貼り付けて接続）を行うと、同じスプレッドシートのデータを共有できる。

## コードを更新したとき

`Code.gs` の内容を変更した場合は、スプレッドシート側の Apps Script エディタでコードを貼り替えたあと、
「デプロイ」→「デプロイを管理」→ 既存デプロイの鉛筆アイコン →「新しいバージョン」を選んで再デプロイする。
（URLは変わらないので、アプリ側の設定は変更不要。）

## 注意事項

- このアプリは常にオンラインでスプレッドシートと通信します。オフラインでは記録できません。
- スプレッドシートには `Child` / `Feedings` / `Growth` / `ScheduleCustom` の4つのシートが自動作成されます。手動で編集する場合は列の並びを変えないよう注意してください。
